# L2 Context and Role

## 1. 문서 목적

이 문서는 VSH Layer 2의 위치와 역할을 정의한다.

현재 저장소는 두 가지 축이 공존하는 상태다.

- `VSH_Project_MVP`
  - 현재 실제 개발과 검증이 진행되는 기준선
  - `layer2/` 패키지 아래에 L2 구현이 누적되는 구조
- `src/vsh`
  - 차세대 구조를 검토했던 패키지 경로
  - 현재는 실사용 소스가 아니라 참고 흔적만 남아 있는 상태

Layer 2 개발의 기준선은 아래처럼 둔다.

- 설계 및 구현 기준: `VSH_Project_MVP/layer2`
- 참고 문서 기준: `VSH_Project_MVP`
- 운영 브랜치 기준: `layer2`

즉, 현재 Layer 2는 `VSH_Project_MVP/layer2`를 실제 기준선으로 유지하며 확장한다.

## 2. L2의 위치

L2는 L1과 L3 사이의 중간 계층이다.

- L1은 빠르게 의심 항목을 탐지한다
- L2는 그 항목을 근거 기반으로 보강하고 검증하고 수정 방향으로 연결한다
- L3는 저장소 전체 수준의 분석, SBOM, 리포트 생성 등을 담당한다

## 3. L2의 핵심 책임

현재 브랜치에서 구현할 L2의 핵심 책임은 다음과 같다.

- L1 findings를 입력으로 받는다
- finding별 근거를 보강한다
- 사실 검증이 가능한 항목은 검증 결과를 붙인다
- 수정 patch 또는 diff를 생성한다
- 사람이 검토 가능한 rationale과 recommendation을 반환한다
- 일부 하위 단계가 실패해도 전체 응답은 유지한다

L2는 탐지 계층이 아니라 판단, 근거 보강, 검증, 수정 제안 계층이다.
